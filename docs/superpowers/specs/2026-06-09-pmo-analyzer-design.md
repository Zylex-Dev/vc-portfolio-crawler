# PMO Analyzer — Design Spec

**Date:** 2026-06-09  
**Status:** Approved  

---

## Overview

Batch-pipeline для оценки 3535 стартапов из `data/all_companies.csv` по критериям
Персонализированной Модели Образования (ПМО). Результат — обогащённый CSV с оценками
по 10-балльной шкале для каждого из 5 инструментов ПМО и итоговым `pmo_score`.

---

## Контекст

- **Исходные данные:** `data/all_companies.csv` — 3535 строк, 11 фондов
- **Ключевые поля для анализа:** `description`, `sectors`, `stage`, `website`
- **Фреймворк ПМО:** разработан для российского школьного образования, оценивает,
  насколько продукт стартапа воплощает принципы персонализированного обучения

---

## ПМО Фреймворк

### 5 Инструментов (оцениваемые колонки)

| Колонка | Инструмент | Описание |
|---------|-----------|----------|
| `pmo_traj` | Персонализированная Траектория | Создание и реализация учеником собственного образовательного пути |
| `pmo_mat` | Учебные Материалы | Адаптивный контент под интересы, темп и каналы восприятия ученика |
| `pmo_collab` | Совместная Деятельность | Форматы взаимодействия между участниками образовательного процесса |
| `pmo_game` | Геймификация и Визуализация | Игровые механики и приёмы визуализации |
| `pmo_feedback` | Обратная Связь | Индивидуальная или групповая реакция на действия и результаты |

### 3 Образовательных Результата (контекст для scoring)

- **Картина мира** — системное видение и место человека в реальности
- **Мотивированная деятельность** — инициатива, самоконтроль, целеполагание
- **Навыки будущего** — компетенции для жизни и работы в современном мире

### Шкала оценки (0–10)

| Диапазон | Значение |
|----------|----------|
| 0–2 | Нет признаков инструмента |
| 3–4 | Косвенная или потенциальная связь |
| 5–6 | Элементы присутствуют, не центральные |
| 7–8 | Явная реализация, значимая функция |
| 9–10 | Центральная функция продукта |

`pmo_score` = среднее арифметическое пяти инструментов (округлённое до 1 знака).

## Архитектура

### Структура файлов

```
pmo_analyzer/
  __init__.py
  scraper.py        # Шаг 1: async скрапинг сайтов
  batch_submit.py   # Шаг 2: формирование и отправка Batches API запроса
  batch_collect.py  # Шаг 3: получение результатов и обогащение CSV
  prompt.py         # PMO system prompt + построитель user-промптов

data/
  all_companies.csv         # исходный файл (не изменяется)
  scraped.json              # артефакт шага 1
  batch_id.txt              # артефакт шага 2
  all_companies_pmo.csv     # финальный результат (шаг 3)
```

### Pipeline (3 шага)

```
python -m pmo_analyzer.scraper        # ~30 мин, создаёт data/scraped.json
python -m pmo_analyzer.batch_submit   # ~1 мин,  создаёт data/batch_id.txt
python -m pmo_analyzer.batch_collect  # ~1 час,  создаёт data/all_companies_pmo.csv
```

## Компоненты

### scraper.py

- `httpx.AsyncClient` + `BeautifulSoup4`
- 50 параллельных запросов (`asyncio.Semaphore(50)`)
- Таймаут: 10 секунд на запрос
- Извлекаем: `<title>`, `<meta description>`, видимый `<body>` текст — первые **3000 символов**
- **Фолбэк:** при любой ошибке (JS-only, 403, timeout, DNS fail) `scraped_text = ""`
- Прогресс: `tqdm` по числу обработанных URL
- Результат: `data/scraped.json` — `{"slug": "scraped_text", ...}`
- **Идемпотентность:** пропускает slug'и, уже присутствующие в `scraped.json`
  (безопасно перезапускать при прерывании)

### prompt.py

**System prompt** содержит:
- Описание каждого из 5 инструментов ПМО с примерами признаков
- Шкалу 0–10 с критериями для каждого диапазона
- Инструкцию отвечать только валидным JSON без markdown

**User prompt** на каждый стартап:
```
Name: {name}
Sectors: {sectors}
Stage: {stage}
Description: {description}
Website content: {scraped_text or "N/A"}
```

**Ожидаемый ответ (строгий JSON):**
```json
{
  "pmo_score": 7.2,
  "traj": 8,
  "mat": 9,
  "collab": 5,
  "game": 7,
  "feedback": 8,
  "notes": "Адаптивный контент по профилю — core feature. Нет геймификации."
}
```

### batch_submit.py

- Читает `data/all_companies.csv` + `data/scraped.json`
- Строит список `Request` объектов (один на строку CSV)
- Модель: `claude-haiku-4-5` (классификационная задача, высокий объём)
- `max_tokens`: 256 (достаточно для JSON-ответа)
- Отправляет через `client.messages.batches.create(requests=[...])`
- Сохраняет `batch.id` в `data/batch_id.txt`
- Выводит ссылку для мониторинга в консоли

### batch_collect.py

- Читает `batch_id` из `data/batch_id.txt`
- Опрашивает статус каждые 60 секунд до `processing_status == "ended"`
- Итерирует результаты через `client.messages.batches.results(batch_id)`
- При `result.type == "succeeded"`: парсит JSON из текстового блока
- При `result.type == "errored"`: записывает `pmo_score = -1` как sentinel для ручного ревью
- Мёрджит по `custom_id` (= `id` из CSV) обратно в DataFrame
- Сохраняет `data/all_companies_pmo.csv`

## Выходной формат

Все исходные колонки CSV сохраняются. Добавляются:

| Колонка | Тип | Пример |
|---------|-----|--------|
| `pmo_score` | float | `7.2` |
| `pmo_traj` | int | `8` |
| `pmo_mat` | int | `9` |
| `pmo_collab` | int | `5` |
| `pmo_game` | int | `7` |
| `pmo_feedback` | int | `8` |
| `pmo_notes` | str | `"Адаптивный контент — core. Нет геймификации."` |

Строки с `pmo_score == -1` — ошибки батча, требуют ручной проверки.

## Ограничения и допущения

- Сайты с JS-only рендерингом (React/Next.js SPA) вернут пустой скрап — фолбэк на description
- Batches API: до 100,000 запросов и 256 MB на батч — наши 3535 укладываются с запасом
- Результаты батча хранятся 29 дней — достаточно для повторного запуска `batch_collect.py`
- PMO-оценка отражает потенциал продукта на основе публичной информации, не глубокий аудит
- Стартапы без `description` и без доступного сайта получат низкий score (мало данных)
